# Web QA UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a browser-based full question-answering interface for the existing Graph RAG agent.

**Architecture:** Add a small standard-library HTTP server in `graph_rag_app.web_ui` that serves static assets and exposes `POST /api/ask`. The API validates JSON input, delegates answer generation to existing `run_or_resume(...)`, and returns structured JSON for the browser.

**Tech Stack:** Python standard library `http.server`, existing Graph RAG runtime functions, vanilla HTML/CSS/JavaScript, `unittest` with `unittest.mock`.

---

## File Structure

- Create `graph_rag_app/web_ui.py`: UI server, API payload parsing, answer execution, static file serving.
- Create `graph_rag_app/static/index.html`: Main question-answer page.
- Create `graph_rag_app/static/styles.css`: Responsive page styling.
- Create `graph_rag_app/static/app.js`: Form submission and answer rendering.
- Modify `graph_rag_app/cli.py`: Add `ui` subcommand and dispatch.
- Modify `graph_rag_app/__init__.py`: Export `serve_ui`.
- Create `tests/test_web_ui.py`: Unit tests for API behavior and CLI integration.
- Modify `README.md`: Add UI launch instructions.
